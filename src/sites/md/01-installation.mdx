## Installation

<section>

In der Informatik braucht es viele verschiedenen Werkzeuge um mit dem Computer
und auch mit Code zu arbeiten. Diese Werkzeuge müssen zuerst installiert werden.
Im folgenden Abschnitt finden Sie eine Installationsanleitung für Ihr
Betriebssystem. Stellen Sie sicher dass Sie das richtige Betriebssystem
ausgewählt haben, ansonsten können Sie die Installation garnicht durchführen.

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    ### VSCode

    Als erstes installieren wir VSCode. Das ist unser Editor den wir zum schreiben und editieren von Code verwenden werden. Um VSCode zu installieren, geben Sie den folgenden Befehl ein, und drücken Sie anschliessend auf `Enter`. Stellen Sie sicher das der Befehl richtig geschrieben ist, ansonsten wird es nicht funktionieren.

    ```bash
    brew install --cask visual-studio-code
    ```

    ### Git (Versionierungs-Werkzeug)

    Wenn man Code schreibt, möchte man diesen immer in Versionen verwalten. Dafür gibt es das Werkzeug `git` dass Sie ganz einfach mit dem Befehl hier installieren können. Wir benötigen `git` auch um die Projektvorlagen zu bekommen und auch um diese abzugeben.

    ```bash
    brew install git
    ```

### Dev Container

<section>

[Es kann sein, dass VSCode Sie fragt, ob Sie die Erweiterung `Dev Containers`
installieren möchten. Wenn Sie dort auf ja drücken, können Sie diesen Abschnitt
überspringen] Moderne Webseiten brauchen jeweils einen Webserver um zu
funktionieren. Dafür verwenden wir `Docker` zusammen mit `nodejs`. Damit wir
`Docker` direkt verwenden können, müssen wir eine **Erweiterung** in `VSCode`
installieren. Drücken Sie dafür einfach `CTRL + SHIFT + P` und kopieren Sie den
folgenden Text in das Feld und drücken Sie `Enter`.

```text
ext install ms-vscode-remote.remote-containers
```

## Container

Wenn Sie mit dem Container kommunizieren möchten, müssen Sie das über ein
Terminal machen. Das Terminal kann am Anfang ein wenig einschüchternd wirken, es
ist aber sehr einfach zu bedienen, und auch sehr mächtig. Sie müssen in der
Regel nur einfache Befehle eingeben, und wenn etwas nicht funktionieren sollte,
dann bekommen Sie eine ausführliche Fehlermeldung. Wenn Sie aber alles richtig
geschrieben haben, dann sollte auch alles ohne Probleme funktionieren.

<Example title="Auftrag: Mit dem Container kommunizieren">
Um ein neues Terminal zu öffnen, tippen Sie `CTRL + SHIFT + P` und suchen Sie
nach dem Befehl `Terminal: Create New Terminal`. Dann sollte am unteren Rand ein
neues Terminal aufgehen. Dort können Sie den Befehl `npm run dev` eintippen und
mit `Enter` bestätigen. Das erstellt Ihnen einen neuen Webserver.

Wenn alles richtig funktioniert hat, können Sie über den folgenden Link Ihre
eigene Webseite anzeigen lassen: [http://localhost:5173/](http://localhost:5173)

</Example>

## Webseite anpassen

Bis jetzt haben Sie das Projekt einfach zum laufen gebracht, aber noch nichts
von dem Inhalt geändert. Da es Ihr Projekt ist, soll die Webseite auch etwas
über Sie aussagen.

<Example title="Auftrag: Erste Version">
  Suchen Sie also nach der Datei `index.html` und fügen Sie
  dort Ihren Namen ein. Speichern Sie dann mit `CTRL + S` und schauen Sie sich das
  Resultat im Webbrowser an.
  Herzlichen Glückwunsch! Mit dem Einfügen von Ihrem Namen, haben Sie die erste
  eigene Version von Ihrem Projekt erstellt. Das möchten wir auch so als Version
  festhalten und mithilfe von `git` einen sogenannten `commit` erstellen.
  Fügen Sie als erstes beim `git graph` Ihren Benutzer von Github ein, indem Sie auf das Einstellungssymbol (Zahnrad) drücken.
  Stagen Sie dann die `index.html` Datei. Schreiben Sie eine `commit-message` und drücken Sie auf `Commit`.

</Example>
